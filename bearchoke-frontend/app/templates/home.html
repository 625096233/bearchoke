I wrote this demo for 2 reasons:

  • 1. To write a PoC of the necessary technologies I would need for my next startup and
  • 2. Because I love Spring and the guides they have on their website. It's the best way to learn how to use frameworks.

To those of you who are familiar with the guides on Spring's website or their Sagan reference implementation, this sample is slightly different. The focus is on writing a code base for a real-world scenario:

  • More code than just the code required to demo one feature. As coders we need to integrate with many different persistence stores, message queues, frameworks, versions and more.
  • More samples showcasing the push towards Single Page Applications (SPAs) while talking to a complex back-end. (Gunnar Hillert of Spring has another sample on Github)
  • Different examples on how to persist and query data from different types of DBs
  • Showcasing a Spring Security setup with different AuthenticationManagers based on different persistence stores and URL patterns that can act as a good start for any new application.
  • Showcasing the beauty of Spring's Java Config support alongside Spring Profiles and the Servlet 3 spec.
  • Moving towards truly scalable, event-driven software by leveraging Command Query Responsibility Segregation (CQRS) through the awesome Axon Framework and its great Spring support as well as Spring's own support for the WebSocket protocol.
  • Finally, because this application is hosted on the fantastic Cloud Foundry PaaS, you can see what's required of your application to easily deploy to CF.
